JSON Schema is a rulebook for your JSON data.
It defines how the data should look â€” what keys are allowed, what data types (string, number, boolean, etc.), and whether fields are required.

Example:

  "age": 25,
  "email": "ankit@test.com"
}


JSON Schema for this:

{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer" },
    "email": { "type": "string", "format": "email" }
  },
    "required": ["name", "age", "email"]
}
